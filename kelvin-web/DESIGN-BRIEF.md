## 1. О проекте

**Kelvin** — нативный мультитул для macOS в строке меню.

Одно приложение заменяет десяток утилит: мониторинг энергии и вентиляторов,
быстрые переключатели (Wi-Fi, Bluetooth, Night Shift, тёмная тема),
авто-раскладка клавиатуры RU↔EN, приватность и фаервол, инструменты
разработчика.

- **Платформа:** macOS 11+ (Intel и Apple Silicon)
- **Бизнес-модель:** Freemium. Мониторинг — бесплатно навсегда.
  Pro — разовая покупка $19 (без подписки).
- **Распространение:** напрямую, нотаризованным DMG (не в Mac App Store —
  песочница не позволяет управление железом)
- **Домен:** trykelvin.com
- **Правообладатель:** Artem Balabanov (MEI), Бразилия

## 4. Дизайн-направление

### Референс: **Apple.com вчистую**

- **apple.com/macbook-pro** — как подаётся продукт: огромный заголовок,
  продукт в центре, чистый фон, минимум текста
- **apple.com/control-center** — как показывается UI: крупный скриншот,
  плавные переходы между состояниями
- **apple.com/airpods-pro** — моушн: плавный скролл-рилвил продуктов,
  закреплённые секции (pinned scroll), fade-in/out

### Принципы

1. **Чёрный холст.** Чистый `#000` (не тёмно-синий). Продукт светится на фоне.
2. **Типографика — герой.** SF Pro Display, огромные размеры (до 96px),
   tight letter-spacing. Текст сам по себе дизайн.
3. **Один акцентный цвет.** Бирюзовый `#33C7D1` → синий `#2A86F0`.
   Не радуга. Градиент только на CTA и ключевых акцентах.
4. **Продукт впереди.** Скриншоты и видео приложения — главные элементы
   страницы. Не иконки, не иллюстрации, не абстрактные мокапы.
5. **Воздух.** Большие отступы между секциями. Не теснить.
6. **Плавность.** Все переходы — soft cubic-bezier. Никаких резких движений.

### Цветовая палитра (фиксированная, синхрон с приложением)

```
Акцент:      #33C7D1 (teal)     → CTA, активные элементы
Акцент яркий: #4DEAEA            → хайлайты
Акцент синий: #2A86F0            → вторичный CTA, ссылки
Чёрный:       #000000            → основной фон
Surface:      #1D1D1F            → карточки (Apple product gray)
Текст:        #F5F5F7            → заголовки
Текст muted:  #A1A1A6            → описания
Текст faint:  #6E6E73            → мелочи, legal
```

## 6. Задача для Motion-артиста

### Что нужно анимировать

**Два уровня моушна:**

#### A. Web-анимации (для сайта)

| Элемент | Что делает | Сложность |
|---|---|---|
| **Hero продукт** | При скролле панель плавно появляется, возможно лёгкий parallax | 🔴 |
| **Scroll reveal** | Каждая секция плавно fade-in при попадании во вьюпорт | 🟡 |
| **Features** | При наведении/скролле — подсветка градиентом, плавный hover | 🟡 |
| **Pinned scroll** | Скриншот продукта закреплён, а рядом меняются описания фич | 🔴 |
| **Number counter** | Цена $19 / бесплатно — плавный count-up | 🟢 |
| **Toggles demo** | Анимация переключателей (Wi-Fi, Bluetooth) как в реальном приложении | 🟡 |

#### B. Продуктовое видео/анимация (опционально, отдельно)

| Что | Формат | Назначение |
|---|---|---|
| **Hero loop** | 5-10 сек, зацикленный MP4/WebM | Hero-секция или OG-image |
| **Feature demos** | 3-5 сек каждый, по одной на фичу | В секции Features |
| **App preview** | 15-30 сек, обзор интерфейса | Для шеринга / соцсетей |

### Технические требования

- **Формат:** Lottie (JSON) для UI-анимаций, MP4 (H.264) для видео
- **Вес:** каждая анимация ≤ 200KB (Lottie) / ≤ 2MB (видео)
- **Performance:** анимации не должны блокировать скролл, использовать
  `transform` и `opacity` (не `width`/`height`/`top`)
- **prefers-reduced-motion:** все анимации должны отключаться через CSS media query
